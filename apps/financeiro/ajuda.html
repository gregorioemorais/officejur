SINCRONIZAÇÃO COM GITHUB GIST

Configure seus computadores com segurança

Este guia explica todo o processo: criar a credencial no GitHub, conectar o primeiro navegador, adicionar outros computadores e entender como o sistema protege as alterações de cada pessoa.

Antes de começar

Escolha uma pessoa responsável pela conta do GitHub e pelo Gist. O Gist deve permanecer privado, e o token nunca deve ser enviado por e-mail, WhatsApp ou captura de tela.

VISÃO GERAL

Como a sincronização funciona

O sistema mantém uma cópia dos dados neste navegador e usa um Gist privado como ponto central para compartilhar as informações entre computadores. O Gist é um arquivo protegido dentro da conta do GitHub.

Gist privadoCópia central
Computador BDados locais
  • Ao sincronizar, o sistema primeiro lê o conteúdo atual do Gist.
  • Inclusões, edições e exclusões são mescladas registro por registro.
  • Alterações feitas em computadores diferentes são preservadas.
  • Se o mesmo registro for alterado em dois lugares, prevalece a alteração mais recente.
  • O token fica somente no navegador em que foi configurado; ele não é gravado no Gist.

ETAPA 1

Crie um token de acesso no GitHub

O token funciona como uma senha exclusiva para o sistema acessar o Gist. Recomendamos um token refinado, com acesso apenas a Gists e com prazo de validade.

  1. 1
    Entre na conta responsável

    Acesse o GitHub com a conta que será proprietária do Gist.

    Abrir login do GitHub
  2. 2
    Abra a criação de token refinado

    No GitHub, acesse Settings → Developer settings → Personal access tokens → Fine-grained tokens e escolha Generate new token.

    Criar token refinado
  3. 3
    Defina identificação e validade

    Use um nome que identifique o equipamento, como “Financeiro — notebook recepção”. Escolha uma validade compatível com a política do escritório; 90 dias é uma opção conservadora.

  4. 4
    Conceda somente a permissão necessária

    Em Account permissions, localize Gists e selecione Read and write. O sistema não precisa de acesso de gravação aos repositórios.

  5. 5
    Gere e copie o token

    Clique em Generate token e copie imediatamente. O GitHub mostra o valor completo apenas uma vez.

Alternativa: token clássico

Se a conta não oferecer a permissão refinada para Gists, crie um token clássico e marque somente o escopo gist. Não marque repo nem outras permissões desnecessárias.

Criar token clássico com escopo gist
Documentação oficial

O GitHub exige permissão de usuário “Gists: write” para criar e atualizar Gists por meio de tokens refinados.

Consultar documentação do GitHub

ETAPA 2

Configure o primeiro computador

  1. 1
    Abra as configurações

    No Financeiro Jurídico, clique na engrenagem no canto superior direito.

  2. 2
    Cole o token

    No campo “Token do GitHub”, cole a credencial criada na etapa anterior.

  3. 3
    Confirme o nome do arquivo

    Mantenha financeiro-juridico.json. Todos os computadores devem usar exatamente o mesmo nome.

  4. 4
    Crie o Gist privado

    Deixe o campo “Gist ID” vazio e clique em Criar Gist. O sistema criará o arquivo privado, preencherá o ID automaticamente e enviará os dados deste navegador.

  5. 5
    Guarde o Gist ID

    O ID é a sequência final do endereço do Gist. Exemplo: em gist.github.com/usuario/abc123, o ID é abc123. Ele será usado nos outros computadores.

Configuração concluída

Quando o topo mostrar “Gist configurado”, use o botão “Sincronizar” para confirmar que os dados já estão atualizados.

ETAPA 3

Adicione outro computador ou navegador

Repita esta etapa em cada navegador que terá acesso ao sistema. Perfis diferentes do mesmo navegador também possuem configurações separadas.

  1. 1
    Crie uma credencial para o equipamento

    Por segurança, prefira um token diferente para cada computador. Assim, um equipamento perdido pode ter seu token revogado sem interromper os demais.

  2. 2
    Informe o mesmo Gist ID

    Abra a engrenagem e cole o ID criado no primeiro computador.

  3. 3
    Informe token e nome do arquivo

    Cole o token autorizado e mantenha financeiro-juridico.json.

  4. 4
    Baixe antes de começar a trabalhar

    Clique em Baixar e mesclar. Isso traz os dados existentes do Gist sem apagar eventuais registros locais.

  5. 5
    Salve e sincronize

    Ative a sincronização automática se desejar e clique em “Sincronizar” para confirmar a conexão.

USO DIÁRIO

Boas práticas entre computadores

Ao iniciar

Clique em “Sincronizar” antes de começar, especialmente se outra pessoa utilizou o sistema em outro computador.

Durante o trabalho

Com a opção automática ativada, alterações salvas são enviadas após um pequeno intervalo.

Ao terminar

Clique novamente em “Sincronizar” e aguarde a mensagem de confirmação antes de fechar a página.

Edição simultânea

Evite duas pessoas editarem o mesmo cliente, lançamento ou processo ao mesmo tempo. Para o mesmo registro, prevalece a versão mais recente.

O que cada botão faz

SincronizarLê o Gist, mescla com este navegador e atualiza a cópia central.
Baixar e mesclarTraz o conteúdo do Gist e combina com os dados locais sem enviar imediatamente.
Criar GistCria a primeira cópia privada quando ainda não existe um Gist ID.
Sincronização automáticaAgenda uma sincronização depois de cada alteração salva.

SEGURANÇA

Proteja os dados e as credenciais

  • Use um Gist privado. Nunca transforme o arquivo financeiro em Gist público.
  • Crie tokens com somente a permissão de Gists e, sempre que possível, um token por equipamento.
  • Defina validade e renove o token quando o GitHub indicar expiração.
  • Não compartilhe tokens em mensagens, documentos, capturas de tela ou gerenciadores não autorizados.
  • O token fica no armazenamento local do navegador. Limpar os dados do navegador remove a configuração daquele equipamento.
  • Um Gist privado controla acesso, mas não é criptografia ponta a ponta. A conta do GitHub deve usar senha forte e autenticação em dois fatores.

SOLUÇÃO DE PROBLEMAS

Mensagens e correções mais comuns

“Bad credentials” ou status 401
O token está incorreto, expirou ou foi revogado. Gere um novo token, substitua-o na engrenagem e salve novamente.
Status 403 ou “Resource not accessible”
O token não possui permissão de gravação em Gists. Em um token refinado, confirme Account permissions → Gists → Read and write. Em um token clássico, confirme o escopo gist.
Status 404 ou Gist não encontrado
Confira se o Gist ID foi copiado integralmente e se o token pertence a uma conta com acesso ao Gist privado. Não cole o endereço completo no campo: use somente o ID.
Arquivo JSON não encontrado
Confirme que todos os computadores usam o mesmo nome: financeiro-juridico.json. Evite renomear ou excluir o arquivo diretamente no GitHub.
As alterações não apareceram no outro computador
No computador que fez a alteração, clique em “Sincronizar” e aguarde a confirmação. Depois, sincronize o segundo computador. Verifique também se ambos usam o mesmo Gist ID e nome de arquivo.
O token sumiu após limpar o navegador
Isso é esperado: a credencial fica somente no armazenamento local. Reconfigure o equipamento com o Gist ID e um token autorizado; depois use “Baixar e mesclar”.
Preciso recuperar uma versão anterior
Não edite o JSON manualmente. Abra o Gist no GitHub e consulte o histórico de revisões. Antes de restaurar qualquer versão, preserve uma cópia do estado atual e confirme a operação com a pessoa responsável pelo sistema.

Se o problema persistir, anote a mensagem exibida pelo sistema, o navegador utilizado e a etapa em que ocorreu. Nunca inclua o token na captura de tela.
